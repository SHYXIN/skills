# MISSION.md 格式

`MISSION.md` 放在教学工作区根目录。它记录用户为什么要学这个主题，用来决定讲什么、讲多深、哪些内容先不碰。

## 模板

```md
# Mission: {主题}

## 为什么学
{1-3 句话。写真实目标，不写空泛目标。比如“能看懂并修改项目里的 FastAPI 后台”，比“学习 FastAPI”更好。}

## 学会以后能做到
- {一个可观察的能力}
- {另一个可观察的能力}

## 当前水平
{用户已经会什么、容易卡在哪里。未知就写“待校准”。}

## 约束
- {时间、项目背景、学习偏好、暂时不想碰的复杂内容}

## 暂不学习
- {相邻但当前不重要的主题}
```

## 规则

- 一个工作区只服务一个主要学习目标。
- 目标必须具体。如果用户说不清为什么学，先追问，不急着生成 lesson。
- 目标变化时可以更新，但要先确认，并写一条学习记录。
- 保持简短。`MISSION.md` 是指南针，不是课程大纲。
