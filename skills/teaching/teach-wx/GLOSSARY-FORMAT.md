# GLOSSARY.md 格式

`GLOSSARY.md` 是用户已经掌握的术语表，不是预习词典。只有当用户能正确使用一个术语时，才把它写进去。

## 模板

```md
# {主题} 术语表

## 术语

**检索增强生成（RAG, Retrieval-Augmented Generation）**

白话解释：先从资料库里找相关内容，再把这些内容交给模型回答，目的是减少胡编、让答案更贴近资料。

为什么需要：模型自己的记忆可能过期或不可靠，RAG 把“查资料”这一步显式加进流程。

容易误解：RAG 不是让模型变聪明，而是让模型回答时多了一份外部资料。
```

## 规则

- 第一次记录术语时保留常用英文或缩写。
- 定义要短，但必须能帮助用户下次看懂。
- 避免只写“官方定义”。优先写白话解释和为什么需要。
- 如果多个词指同一个东西，选一个主名称，其他作为别名。
- 用户理解加深后，可以直接改旧定义，不需要保留过时说法。
