# 进度跟踪

## 文件位置

固定路径：

```
~/.socratic-tutor/progress.md
```

Windows 示例：`C:\Users\<用户名>\.socratic-tutor\progress.md`

> 路径在用户主目录下，与 skill 安装位置无关。更新 skill 不会丢失进度。

## 文件格式

```markdown
# 学习进度

> 最后更新：2026-05-18

---

## RAG（检索增强生成）

- **当前层级**：L2
- **最后学习**：2026-05-18
- **本轮概念**：RAG 概念、微调 vs RAG、LoRA 原理

### 子话题进度

| 子话题 | 层级 | 最后接触 | 复习次数 | 下次复习 |
|--------|------|----------|----------|----------|
| RAG 概念与架构 | L2 | 2026-05-18 | 0 | 2026-05-19 |
| LoRA 原理 | L2 | 2026-05-18 | 0 | 2026-05-18 |

## 纠偏记录

| 时间 | 话题 | 从 | 到 | 触发方式 |

```

## 艾宾浩斯复习间隔

| 复习次数 | 距上次学习/复习的时间 | 题目数量 | 题型 |
|----------|----------------------|----------|------|
| 第1次 | 1天后 | 2题 | 基础记忆型 |
| 第2次 | 3天后 | 2题 | 理解型 |
| 第3次 | 7天后 | 3题 | 应用型 |
| 第4次 | 15天后 | 2题 | 综合型 |
| 第5次 | 30天后 | 1题 | 挑战型 |

复习次数达到 5 次后标记为"已巩固"，不再安排复习。

## 复习结果判定

- **全部通过**：复习次数 +1，下次复习日期 = 今天 + 对应间隔
- **部分通过（≥50%）**：复习次数 +1，下次复习间隔减半
- **未通过（<50%）**：层级降 1 级，复习次数清零，下次复习日期 = 明天

## 读写规则

### 读取（每次教学开始前）

1. 用 Read 工具读取 `~/.socratic-tutor/progress.md`（固定路径）
2. 如果文件不存在 → 创建（只有 header，没有话题记录）
3. 如果文件存在 → 找到当前话题对应的条目，提取层级作为初始假设
4. **检查复习计划**：遍历所有子话题的"下次复习"字段，如果有到期未复习的，提示用户先复习

### 写入（每次教学交互的保存步骤）

1. 更新当前话题的层级（如有变化）
2. 更新最后学习日期
3. 追加本轮覆盖的概念
4. 更新子话题进度表
5. 如有纠偏，追加纠偏记录
6. **为新学子话题设置复习计划**：复习次数=0，下次复习日期=明天
7. **如果刚完成复习**：复习次数+1，重新计算下次复习日期
8. 用 Write 工具写回同一个文件

### 创建新话题

当用户开始一个全新话题时：

1. 在文件中追加新话题条目
2. 初始层级设为 L1（除非探测显示更高）
3. 子话题表留空，随学习推进填充

## 话题粒度规则

- 用户说"教我机器学习" → 创建"机器学习"话题
- 深入讲"神经网络"时 → 在"机器学习"下记录子话题"神经网络"
- 用户说"换个话题" → 创建新话题

## 层级更新规则

- **默认不变**：理解检测通过时层级不自动升级，等用户表现出更深理解
- **谨慎向下**：只有理解检测失败或用户明确要求时才降级
- **一次一级**：每次纠偏只调整一个层级
- **显式优先**：用户说"太深了"直接降级，不重新探测

## 文件维护

- 纠偏记录保留最近 20 条
- 子话题超过 20 个时删除最久未接触的
- 话题超过 30 天未接触，标注"可能需要重新校准"
- 已巩固的知识点（复习 5 次通过）移入单独的"已巩固"区块
