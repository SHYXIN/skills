# 进度跟踪

## 文件位置

固定路径，不再动态构造：

```
~/.socratic-tutor/progress.md
```

Windows 完整示例：
```
C:\Users\<用户名>\.claude\skills\socratic-tutor\progress\progress.md
```

## 读写规则

### 读取（每次教学开始前）

1. 用 Read 工具读取 `~/.socratic-tutor/progress.md`（绝对路径）
2. 如果文件不存在 → 创建（只有 header，没有话题记录）
3. 如果文件存在 → 找到当前话题对应的条目，提取层级作为初始假设

### 写入（每次教学交互的保存步骤）

1. 更新当前话题的层级（如有变化）
2. 更新最后学习日期
3. 追加本轮覆盖的概念
4. 更新子话题进度表
5. 如有纠偏，追加纠偏记录
6. 用 Write 工具写回同一个文件

### 创建新话题

当用户开始一个全新话题时：

1. 在文件中追加新话题条目
2. 初始层级设为 L1（除非探测显示更高）
3. 子话题表留空，随学习推进填充

## 话题粒度规则

- 用户说"教我机器学习" → 创建"机器学习"话题
- 深入讲"神经网络"时 → 在"机器学习"下记录子话题"神经网络"
- 用户说"换个话题" → 创建新话题

## 层级更新规则

- **默认不变**：理解检测通过时层级不自动升级，等用户表现出更深理解
- **谨慎向下**：只有理解检测失败或用户明确要求时才降级
- **一次一级**：每次纠偏只调整一个层级
- **显式优先**：用户说"太深了"直接降级，不重新探测

## 文件维护

- 纠偏记录保留最近 20 条
- 子话题超过 20 个时删除最久未接触的
- 话题超过 30 天未接触，标注"可能需要重新校准"
