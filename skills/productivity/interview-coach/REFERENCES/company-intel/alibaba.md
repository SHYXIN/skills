# 阿里巴巴 面试情报

## 面试风格

- **八股文 + 实战并重**：既考基础概念，也考实际应用
- **业务场景导向**：喜欢结合实际业务场景出题
- **深挖原理**：不满足于"会用"，要"懂原理"
- **JVM/中间件**：后端岗位几乎必问
- **项目经历追问**：会基于你的简历深挖项目细节

## 各轮次重点

| 轮次 | 重点 | 说明 |
|------|------|------|
| 一面 | 基础 + 编码 | Java 基础 + 1-2 道算法 |
| 二面 | 深度 + 系统设计 | 中间件原理 + 系统设计 |
| 三面 | 架构 + 业务 | 系统架构 + 业务理解 |
| 交叉面 | 广度 + 潜力 | 跨团队面试官，考察综合能力 |
| HR 面 | 价值观 + 动机 | 阿里味、团队合作 |

## 高频考点

### Java 八股（必问）
- HashMap 原理（扩容、红黑树、线程安全）
- ConcurrentHashMap 实现
- JVM 内存模型 + GC 算法
- synchronized vs ReentrantLock
- ThreadLocal 原理和内存泄漏
- volatile 关键字

### 中间件（高频）
- RocketMQ/Kafka 消息可靠性
- Redis 数据结构 + 集群方案 + 缓存三大问题
- MySQL 索引原理 + 事务隔离级别 + MVCC
- Dubbo/Spring Cloud 微服务

### 系统设计
- 秒杀系统
- 分布式事务
- 高并发限流
- 海量数据处理

### 算法
- 难度中等（LeetCode Medium 为主）
- 常考：链表、二叉树、动态规划、排序

## 文化关键词

- **客户第一**
- **团队合作**
- **诚信**
- **激情**
- **敬业**

## 面试官喜欢的回答特征

1. **基础扎实** — 八股文能答清楚
2. **有业务思维** — 能联系实际业务场景
3. **懂原理** — 不只知道怎么用，还知道为什么
4. **有阿里味** — 回答中体现客户导向、团队合作

## 面试官不喜欢的回答特征

1. 八股文答不上来
2. 只会用不懂原理
3. 项目经历说不清楚
4. 过度关注技术细节，忽略业务价值
