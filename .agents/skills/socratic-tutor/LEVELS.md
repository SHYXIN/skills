# 五级认知模型

## 总览

| 层级 | 名称 | 核心特征 | 一句话描述 |
|------|------|----------|------------|
| L1 | 完全陌生 | 没有相关概念，靠日常经验理解 | "我从来没接触过这个" |
| L2 | 有概念 | 听说过，知道大概是什么，但说不清楚 | "我知道有这么回事，但说不上来" |
| L3 | 有基础 | 理解基本原理，能简单应用，但不成体系 | "我懂基础，但细节不清楚" |
| L4 | 有经验 | 实践过，理解底层逻辑，能处理复杂情况 | "我用过/研究过，想深入" |
| L5 | 精通 | 能教学、能批判、能看到领域边界 | "我想教别人/探索前沿" |

---

## L1：完全陌生

### 检测信号
- 问"X 是什么意思？"（连基本定义都不知道）
- 用完全不相关的领域来类比
- 对任何术语都表现出陌生感
- 回答非常简短或沉默

### 讲解策略

| 维度 | 策略 |
|------|------|
| **术语密度** | 零术语。只用日常语言。必须使用专业术语时，立即用括号解释 |
| **抽象层级** | 只讲"是什么"和"有什么用"。不讲原理，不讲为什么 |
| **前置知识假设** | 假设读者只知道日常生活常识。不假设任何领域知识 |
| **例子复杂度** | 只用生活中的常见场景：做饭、购物、交通、社交 |

### 典型表达
> "机器学习就像教小孩认猫——你给他看很多猫的照片，他就慢慢学会了什么是猫。"

### 理解检测方式
- 让用户举一个自己的例子
- 问"你觉得这个和 [日常事物] 像在哪里？"

---

## L2：有概念

### 检测信号
- 能说出基本定义，但不够准确
- 用"好像"、"大概"、"应该是"等模糊词
- 能举一个例子，但举不好
- 混淆相关概念（如分不清"机器学习"和"深度学习"）

### 讲解策略

| 维度 | 策略 |
|------|------|
| **术语密度** | 引入核心术语，每个术语第一次出现时用类比解释。不超过 3 个新术语/次 |
| **抽象层级** | 讲"是什么"和"怎么运作"的简化版本。开始涉及"为什么"，但用直觉方式解释 |
| **前置知识假设** | 假设读者有 L1 的基础知识。可以引用 L1 的类比作为桥梁 |
| **例子复杂度** | 用领域内简单例子 + 生活类比。开始引入简化版真实场景 |

### 典型表达
> "机器学习是让计算机从数据中自动发现规律。比如邮件系统看了很多垃圾邮件后，就能自动识别新的垃圾邮件——它不是靠人工写规则，而是自己'学'出来的。"

### 理解检测方式
- 让用户用自己的话复述
- 问"你能想到另一个用这个原理的例子吗？"

---

## L3：有基础

### 检测信号
- 能准确使用基本术语
- 能解释基本原理，但遇到细节就模糊
- 知道"是什么"，但对"为什么这样设计"不清楚
- 能完成简单任务，但遇到变体就卡住

### 讲解策略

| 维度 | 策略 |
|------|------|
| **术语密度** | 正常使用术语，不再逐一解释基础术语。引入高级术语时简要说明 |
| **抽象层级** | 讲原理、讲设计决策、讲"为什么这样而不是那样"。开始涉及边界条件和常见误区 |
| **前置知识假设** | 假设读者掌握 L2 全部内容。可以直接引用基础概念而不解释 |
| **例子复杂度** | 用真实场景的例子。展示常见误区和如何避免 |

### 典型表达
> "监督学习需要标注数据——你给模型输入和正确答案，它学习两者之间的映射。但这里有个陷阱：如果训练数据有偏见，模型也会学到偏见。比如招聘系统如果历史数据中男性候选人更多，模型就会偏向推荐男性。"

### 理解检测方式
- 问"如果 [某个条件] 变了，结果会怎样？"
- 问"这个方法在什么情况下会失效？"

---

## L4：有经验

### 检测信号
- 能准确、流畅地使用专业术语
- 能讨论不同方法的优缺点
- 提出的问题涉及细节、边界情况、性能
- 能指出某个说法的不准确之处

### 讲解策略

| 维度 | 策略 |
|------|------|
| **术语密度** | 全术语。使用领域黑话和缩写，不再解释。可以引入跨领域术语做类比 |
| **抽象层级** | 讲底层原理、讲设计权衡、讲不同学派的争论。讨论"为什么这个方案赢了/输了" |
| **前置知识假设** | 假设读者掌握 L3 全部内容，有实践经验。可以直接讨论高级话题 |
| **例子复杂度** | 用真实项目案例、论文结果、行业数据。讨论失败案例和教训 |

### 典型表达
> "Transformer 的核心是 self-attention，它让模型在处理每个词时能'看到'句子中所有其他词。相比 RNN 的顺序处理，这解决了长距离依赖问题，但代价是 O(n²) 的计算复杂度——这就是为什么处理长文档时人们要用 sparse attention 或者 sliding window 来优化。"

### 理解检测方式
- 问"你觉得这个方案最大的局限是什么？"
- 问"如果让你设计一个替代方案，你会怎么做？"
- 让用户评价两种方法的优劣

---

## L5：精通

### 检测信号
- 能指出当前讲解中的不精确之处
- 提出的问题涉及前沿、未解决问题、学术争议
- 能从多个角度分析同一个概念
- 表达方式像是在准备教学或写论文

### 讲解策略

| 维度 | 策略 |
|------|------|
| **术语密度** | 全术语 + 元语言。讨论术语本身的演变和争议。引用论文和学者 |
| **抽象层级** | 讲哲学、讲范式、讲这个领域和其他领域的深层联系。讨论"我们为什么这样思考"以及"有没有其他思考方式" |
| **前置知识假设** | 假设读者掌握 L4 全部内容，可能有自己的研究方向。可以讨论未发表的想法 |
| **例子复杂度** | 用反例、悖论、开放问题。讨论"我们不知道什么" |

### 典型表达
> "其实'注意力'这个概念本身是有争议的。self-attention 真的是在'注意'吗？还是说它本质上是一种可微的寻址机制？Bengio 学派和 Schmidhuber 学派在这个问题上有根本分歧——这不只是语义之争，它影响着你如何设计下一代架构。"

### 理解检测方式
- 问"你觉得这个领域最大的未解问题是什么？"
- 问"如果你是这个领域的开创者，你会走什么不同的路？"
- 让用户教回来——"你会怎么向另一个专家解释这个？"

---

## 层级调整速查

当需要调整层级时，参考以下方向：

### 降低一层时
- 换一个更生活化的类比
- 去掉术语，换成大白话
- 只讲"是什么"，不讲"为什么"
- 把一个概念拆成更小的步骤

### 提升一层时
- 加入术语和精确定义
- 讲设计决策和权衡
- 引入边界条件和反例
- 联系到更大的知识体系

---

## 常见领域的典型进阶路径

### 编程/计算机科学
```
变量/数据类型 → 控制流 → 函数 → 数据结构 → 算法 → 设计模式 → 系统架构 → 分布式系统
```

### 机器学习
```
什么是ML → 监督/非监督 → 损失函数/优化 → 神经网络 → CNN/RNN → Transformer → 预训练/微调 → 前沿研究
```

### 经济学
```
供给/需求 → 市场均衡 → 弹性 → 宏观指标 → 货币政策 → 博弈论 → 行为经济学 → 前沿理论
```

### 语言学习
```
发音/拼音 → 基础词汇 → 基本语法 → 日常对话 → 阅读写作 → 文化理解 → 高级表达 → 母语级运用
```

### 数学
```
算术 → 代数 → 几何 → 三角/解析几何 → 微积分 → 线性代数 → 概率统计 → 实分析/抽象代数
```

### 历史
```
时间线/大事件 → 因果关系 → 社会结构分析 → 史学方法论 → 比较历史 → 历史哲学
```

> 注意：以上路径仅供参考。实际教学中应根据用户的具体背景和兴趣灵活调整。
> 不是每个领域都需要严格按路径走——有些用户可能只想深入某个子话题。
