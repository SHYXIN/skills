# 进度跟踪

## 用户标识

用户标识由以下部分组成：

```
<项目名>_<用户名>
```

- **项目名**：当前 Claude Code 工作区的目录名（如 `my-project`）
- **用户名**：从系统环境变量 `USERNAME` 或 `USER` 获取

示例：`ai-research_WX847`

如果无法获取上述信息，使用 `default` 作为标识。

## 文件位置

```
socratic-tutor/progress/<用户标识>.md
```

示例：`socratic-tutor/progress/ai-research_WX847.md`

## 读写规则

### 读取（每次教学开始前）

1. 根据用户标识构造文件路径
2. 如果文件不存在，创建文件（只有 header，没有话题记录）
3. 如果文件存在，找到当前话题对应的条目
4. 提取当前层级作为初始假设
5. 提取子话题层级，用于判断用户在该话题下的知识分布

### 写入（每次教学交互后）

1. 更新当前话题的层级
2. 更新最后学习日期
3. 追加本轮覆盖的概念
4. 更新子话题进度表
5. 如果有纠偏，追加纠偏记录

### 创建新话题

当用户开始一个全新的话题时：

1. 在文件中追加新话题条目
2. 初始层级设为 L1（除非探测显示更高）
3. 子话题表留空，随学习推进填充

## 话题粒度规则

**存到多细？**

- 用户说"教我机器学习" → 创建"机器学习"话题
- 深入讲"神经网络"时 → 在"机器学习"下记录子话题"神经网络"
- 用户说"换个话题，讲讲法国历史" → 创建新话题"法国历史"

**什么时候创建新话题 vs. 子话题？**

- 用户明确说"换个话题" → 新话题
- 用户在同一领域深入某个方向 → 子话题
- 判断模糊时 → 优先创建子话题（可以在对话中确认）

## 层级更新规则

- **默认向上**：如果理解检测通过，层级保持不变（不要自动升级，等用户表现出更深理解）
- **谨慎向下**：只有当理解检测失败或用户明确要求时才降级
- **一次一级**：每次纠偏只调整一个层级，不跨级跳
- **显式优先**：如果用户说"太深了"，直接降级，不要重新探测

## 文件维护

- 纠偏记录保留最近 20 条，超出则删除最旧的
- 子话题超过 20 个时，删除最久未接触的
- 话题超过 30 天未接触，标注为"可能需要重新校准"
